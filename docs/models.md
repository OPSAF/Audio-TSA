# 深度学习模型详解

本文档详细介绍 Audio Lab 中使用的深度学习模型，包括网络架构、训练策略、数据划分方式和预测流程。

---

## 一、模型概述

Audio Lab 集成了四种预测模型，覆盖经典统计方法到现代深度学习：

| 模型 | 类型 | 特点 | 适用场景 |
|------|------|------|----------|
| **ARIMA** | 统计模型 | 理论成熟、可解释性强 | 短期预测、平稳序列 |
| **HMM** | 概率模型 | 处理隐状态、状态转移 | 状态切换、模式识别 |
| **LSTM** | 深度学习 | 长期记忆、门控机制 | 长序列、复杂模式 |
| **Transformer** | 深度学习 | 自注意力、并行计算 | 长序列、全局依赖 |

## 二、数据准备与划分

### 2.1 输入数据

**原始数据来源：**
- Mel 语谱图（Mel Spectrogram）
- 形状：`(n_mels, n_frames)`，通常 `(128, T)`

**数据预处理流程：**

```
音频信号 → STFT → 功率谱 → Mel滤波器组 → 对数压缩 → Mel语谱图
```

**为什么使用 Mel 语谱图？**
1. 降维：从时域波形降到 128 个频带
2. 语义：Mel 刻度符合人耳感知
3. 稳定：对数压缩减小动态范围

### 2.2 数据划分策略

**划分比例：**
- **训练集**：70%
- **验证集**：15%
- **测试集**：15%

**划分方式：时间序列划分**

```
|←───── 训练集 (70%) ─────→|←── 验证集 (15%) ──→|←── 测试集 (15%) ──→|
```

**重要原则：**
- **禁止随机打乱**：时间序列必须保持时间顺序
- **禁止数据泄露**：测试集数据不能用于任何训练决策
- **验证集用途**：超参数调优、早停判断

### 2.3 序列构建

**滑动窗口方式构建训练样本：**

对于序列 $x_1, x_2, \ldots, x_T$，使用窗口长度 $L$：

| 输入序列 | 目标 |
|----------|------|
| $x_1, x_2, \ldots, x_L$ | $x_{L+1}$ |
| $x_2, x_3, \ldots, x_{L+1}$ | $x_{L+2}$ |
| $\vdots$ | $\vdots$ |
| $x_{T-L}, \ldots, x_{T-1}$ | $x_T$ |

**默认窗口长度：** 20 帧

### 2.4 数据标准化

**Z-score 标准化：**

$$x' = \frac{x - \mu}{\sigma}$$

其中 $\mu$ 和 $\sigma$ 仅在训练集上计算，然后应用到验证集和测试集。

## 三、ARIMA 模型

### 3.1 模型原理

ARIMA(p, d, q) 由三部分组成：

- **AR(p)**：自回归部分
- **I(d)**：差分部分
- **MA(q)**：移动平均部分

**数学表达式：**

$$(1 - \phi_1 B - \cdots - \phi_p B^p)(1-B)^d x_t = (1 + \theta_1 B + \cdots + \theta_q B^q) \epsilon_t$$

### 3.2 自动定阶

使用 AIC（赤池信息准则）自动选择最优阶数：

$$AIC = 2k - 2\ln(\hat{L})$$

其中：
- $k$：参数数量
- $\hat{L}$：最大似然估计值

**搜索范围：**
- p ∈ [0, 5]
- d ∈ [0, 2]
- q ∈ [0, 5]

### 3.3 预测流程

```
1. 检验平稳性（ADF检验）
2. 若非平稳，进行差分
3. AIC自动定阶
4. 拟合模型参数
5. 滚动预测未来h步
```

### 3.4 优缺点分析

**优点：**
- 理论成熟，可解释性强
- 对线性模式效果好
- 计算效率高

**缺点：**
- 难以捕捉非线性模式
- 对长序列建模能力有限
- 需要平稳性假设

## 四、隐马尔可夫模型（HMM）

### 4.1 模型原理

HMM 假设系统存在隐状态序列，观测值由隐状态生成：

$$P(O|λ) = \sum_{all Q} P(O|Q,λ) P(Q|λ)$$

**三要素：**
- **初始概率 $\pi$**：各状态的初始概率
- **转移概率 $A$**：状态转移矩阵
- **发射概率 $B$**：各状态的观测分布

### 4.2 模型配置

**默认参数：**
- 隐状态数：3
- 协方差类型：对角协方差
- 迭代次数：100

**状态数选择：**
- 2 状态：简单模式（如安静/活动）
- 3 状态：中等复杂度（默认）
- 5 状态：复杂模式

### 4.3 训练算法

**Baum-Welch 算法（EM算法）：**

1. **E步**：计算前向概率 $\alpha$ 和后向概率 $\beta$
2. **M步**：更新参数 $\pi, A, B$
3. 重复直到收敛

### 4.4 预测方法

使用维特比算法解码最可能的状态序列，然后基于状态转移预测未来：

$$\hat{s}_{t+1} = \arg\max_s P(s|s_t)$$

$$\hat{o}_{t+1} = \mathbb{E}[o|s_{t+1}]$$

### 4.5 优缺点分析

**优点：**
- 可建模状态切换
- 概率框架，可给出不确定性估计
- 对分段数据效果好

**缺点：**
- 马尔可夫假设限制了长程依赖
- 高斯发射假设可能不适用
- 状态数需要手动设定

## 五、LSTM 模型

### 5.1 网络架构

**LSTM 单元结构：**

```
        ┌─────────────────────────────────────┐
        │              LSTM Cell              │
        │                                     │
c_t ────┼──→ [  遗忘门  ] ──→ c_{t-1} × f_t ──┼──→ c_t (细胞状态)
        │         ↓                           │
        │    [  输入门  ] ──→ i_t × tanh(W·x) │
        │                                     │
h_t ────┼──→ [  输出门  ] ──→ o_t × tanh(c_t)─┼──→ h_t (隐藏状态)
        │                                     │
        └─────────────────────────────────────┘
```

**门控机制：**

$$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$$  （遗忘门）

$$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$$  （输入门）

$$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$$  （输出门）

$$\tilde{c}_t = \tanh(W_c \cdot [h_{t-1}, x_t] + b_c)$$  （候选细胞）

$$c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$$  （细胞状态更新）

$$h_t = o_t \odot \tanh(c_t)$$  （隐藏状态输出）

### 5.2 网络配置

**默认超参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 隐藏层大小 | 64 | LSTM隐藏状态维度 |
| 层数 | 2 | 堆叠LSTM层数 |
| Dropout | 0.2 | 防止过拟合 |
| 学习率 | 0.001 | Adam优化器 |
| 批大小 | 32 | 每批样本数 |
| 训练轮数 | 50 | 最大训练轮数 |

### 5.3 训练流程

```
1. 构建训练数据集（滑动窗口）
2. 划分训练/验证集（70%/30%）
3. 初始化网络和优化器
4. For epoch in range(max_epochs):
   a. 前向传播计算损失
   b. 反向传播更新参数
   c. 在验证集上评估
   d. 若验证损失连续5轮不下降，早停
5. 返回最佳模型
```

### 5.4 早停策略

**实现方式：**
- 监控验证集损失
- 耐心值（patience）：5 轮
- 若连续 5 轮验证损失不下降，停止训练
- 返回验证损失最小的模型

**目的：** 防止过拟合，提高泛化能力

### 5.5 预测方法

**滚动预测：**

```python
# 自回归预测
predictions = []
current_input = last_sequence  # 最后L个观测值

for _ in range(forecast_horizon):
    pred = model(current_input)
    predictions.append(pred)
    current_input = torch.cat([current_input[:, 1:, :], pred.unsqueeze(1)], dim=1)
```

### 5.6 优缺点分析

**优点：**
- 长期记忆能力强
- 能捕捉复杂非线性模式
- 门控机制有效控制信息流

**缺点：**
- 训练时间较长
- 需要大量数据
- 超参数调优复杂

## 六、Transformer 模型

### 6.1 网络架构

**Transformer 编码器结构：**

```
Input → Positional Encoding → Multi-Head Attention → Feed Forward → Output
                                    ↑                    ↑
                              Add & Norm          Add & Norm
```

**自注意力机制：**

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

其中：
- $Q = XW_Q$：查询矩阵
- $K = XW_K$：键矩阵
- $V = XW_V$：值矩阵
- $d_k$：缩放因子，防止点积过大

**多头注意力：**

$$\text{MultiHead}(Q, K, V) = \text{Concat}(head_1, \ldots, head_h) W_O$$

### 6.2 位置编码

由于 Transformer 没有循环结构，需要显式编码位置信息：

$$PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{model}})$$

$$PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{model}})$$

### 6.3 网络配置

**默认超参数：**

| 参数 | 默认值 | 说明 |
|------|--------|------|
| d_model | 64 | 模型维度 |
| n_heads | 4 | 注意力头数 |
| n_layers | 2 | 编码器层数 |
| d_ff | 256 | 前馈网络维度 |
| Dropout | 0.1 | Dropout率 |
| 学习率 | 0.001 | Adam优化器 |
| 批大小 | 32 | 每批样本数 |
| 训练轮数 | 50 | 最大训练轮数 |

### 6.4 训练策略

**学习率调度：**

使用余弦退火学习率：

$$\eta_t = \eta_{min} + \frac{1}{2}(\eta_{max} - \eta_{min})(1 + \cos(\frac{t}{T}\pi))$$

**梯度裁剪：**

防止梯度爆炸：

$$\nabla' = \begin{cases} \nabla & \|\nabla\| \leq c \\ \frac{c}{\|\nabla\|} \nabla & \|\nabla\| > c \end{cases}$$

默认裁剪阈值：1.0

### 6.5 优缺点分析

**优点：**
- 并行计算能力强
- 能捕捉长程依赖
- 注意力权重可解释

**缺点：**
- 对短序列可能过拟合
- 计算复杂度 $O(n^2)$
- 需要位置编码

## 七、模型对比与选择

### 7.1 性能对比

| 指标 | ARIMA | HMM | LSTM | Transformer |
|------|-------|-----|------|-------------|
| 短期预测 | ★★★★ | ★★★ | ★★★ | ★★★ |
| 长期预测 | ★★ | ★★ | ★★★★ | ★★★★ |
| 训练速度 | ★★★★★ | ★★★★ | ★★ | ★★ |
| 可解释性 | ★★★★★ | ★★★★ | ★★ | ★★★ |
| 非线性建模 | ★ | ★★ | ★★★★ | ★★★★ |

### 7.2 选择建议

**场景一：快速分析**
- 推荐：ARIMA
- 原因：训练快，可解释

**场景二：状态切换明显**
- 推荐：HMM
- 原因：可建模隐状态

**场景三：复杂长序列**
- 推荐：LSTM 或 Transformer
- 原因：长期记忆能力强

**场景四：追求最佳精度**
- 推荐：四模型对比，选择最优
- 原因：不同数据特性不同

## 八、思考与改进方向

### 8.1 当前局限

1. **数据量限制**：深度学习模型在小数据集上容易过拟合
2. **超参数调优**：目前使用固定超参数，未进行系统调优
3. **集成学习**：未充分利用多模型集成的优势
4. **不确定性量化**：预测结果缺少置信区间

### 8.2 改进方向

1. **数据增强**：
   - 时间拉伸、音高变换
   - 添加噪声增强鲁棒性

2. **超参数优化**：
   - 贝叶斯优化
   - 网格搜索

3. **模型集成**：
   - 加权平均
   - Stacking

4. **概率预测**：
   - Bayesian LSTM
   - MC Dropout

### 8.3 未来工作

- 引入预训练模型（如 Wav2Vec）
- 多任务学习（预测 + 分类）
- 在线学习（增量更新）
