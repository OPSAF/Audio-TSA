# Audio Lab 项目说明

## 研究背景

Audio Lab 是一个面向音频信号的时间序列分析系统。它不是简单地把音频当成“声音文件”，而是把每一段波形视为一条时间序列，提取其动态趋势、时序结构、波动特征和可预测性。

在声音信号中，信息来源于振幅变化、频率成分、能量分布、周期结构以及这些量随时间的演化。相比传统的“特征提取工具箱”，本项目的核心在于：

- 将声音转成可以直接分析的时间序列；
- 用时间序列分析方法理解音频中的结构与波动；
- 评估音频特征的可预测性，而不是只做一次性识别；
- 在双音频对比中寻找趋势、波动和结构上的对应关系。

## 时间序列构造

### 为什么音频适合作为时间序列分析对象

音频从本质上就是一个随时间变化的信号：

- 每个采样点代表一个瞬时振幅；
- 样本按时间顺序排列；
- 相邻样本之间存在强关联；
- 不同时间尺度上包含不同的结构：瞬态、周期、音乐片段、情绪变化。

因此，音频天然满足时间序列分析的两个核心条件：

1. **时间顺序性**。音频采样序列按时间递增排列。
2. **动态相关性**。当前样本与过去样本具有依赖关系。

### 从音频到时间序列的第一步

本项目中，音频文件首先被读取为数字信号：

```python
y, sr = loader.load_audio(filepath, target_sr=16000)
```

其中：

- `y` 是一维数组，表示时间序列样本值；
- `sr` 是采样率，表示每秒采样点个数；
- 统一使用 `16000 Hz`，兼顾语音与音乐分析的精度与计算效率；
- 立体声会转换为单声道，以减少噪声与冗余信息。

### 音频时间序列的本质变换

音频本身是一个原始时域信号，但直接分析完整波形往往难以发现深层结构。Audio Lab 的核心思路是：

- 从原始波形构造多个“描述性时间序列”；
- 让每条时间序列关注一个不同的音乐认知维度；
- 在这些维度上做趋势分析、周期分析、波动分析和可预测性评估。

## 构建时间序列：Energy / Brightness / Complexity / Rhythm

Audio Lab 将音频滑动窗口化，每个窗口生成 4 条关键时间序列。

### 1. Energy 时间序列

Energy 反映的是短时能量强度，也可理解为响度。对窗口中的信号 $x_i$ 计算均方根：

$$E = \sqrt{\frac{1}{N} \sum_{i=1}^{N} x_i^2}$$

其中 $N$ 是窗口大小。Energy 趋势可以识别“高潮/衰减”、“有声/静音”段落。

### 2. Brightness 时间序列

Brightness 以频谱质心衡量音色的明暗：

$$C = \frac{\sum_{k} f_k \cdot M_k}{\sum_{k} M_k}$$

其中 $f_k$ 是第 $k$ 个频率，$M_k$ 是该频率的能量。高 Brightness 说明声音更“明亮”，低 Brightness 说明声音更“暗沉”。

### 3. Complexity 时间序列

Complexity 用频谱熵衡量声音的结构复杂度：

$$H = -\sum_{k} p_k \log p_k, \quad p_k = \frac{M_k}{\sum_j M_j}$$

- 低复杂度：频谱能量集中，信号更规律；
- 高复杂度：频谱分布平坦，信号更加混沌。

Complexity 时间序列可以区分“纯音/和弦”与“噪声/过渡段”。

### 4. Rhythm Density 时间序列

Rhythm Density 反映节奏瞬态的密度与强度。系统使用预加重与频谱通量估计方式，从每个窗口中计算节拍活跃度。某些实现形式为：

$$R = \frac{1}{N-1} \sum_{i=1}^{N-1} |x_{pre}[i]^2 - x_{pre}[i-1]^2|$$

其中 $x_{pre}[i] = x[i] - 0.97x[i-1]$。Rhythm 时间序列擅长揭示节奏断点、鼓点密度和瞬态聚集。

### 为什么使用这四条时间序列

这四条序列分别对应声音的：

- 强度（Energy）
- 频色（Brightness）
- 复杂度与结构性（Complexity）
- 节奏行为（Rhythm）

它们从不同角度描述音频的“动态趋势”，形成了多维时间序列分析的基础。

## 动态趋势分析

### 滑动窗口与时间对齐

系统采用滑动窗口方法：窗口时长 0.5 秒、步长 0.25 秒。这样，每个时间步都有一个新的窗口视图，得到等时长的趋势序列。

这种设计的意义在于：

- 保留足够的时间分辨率；
- 使趋势曲线平滑且可比较；
- 同时避免窗口过窄导致频谱失真。

### 结构分段与片段识别

基于四条趋势曲线，Audio Lab 会对音频进行结构分段：

- 高潮段（Climax）
- 安静段（Calm）
- 积累段（Buildup）
- 释放段（Release）
- 过渡段（Transition）

分段规则基于能源、节奏和复杂度的百分位、导数变化与峰值关系。这个分段结果帮助理解音频的“起承转合”。

## 时间序列分析

### FFT：全局频谱诊断

FFT 用于把一段时间序列转换到频域：

$$X(k) = \sum_{n=0}^{N-1} x(n) e^{-j2\pi kn/N}$$

在音频分析中，FFT 提供了整体频率成分的能量分布，适合回答：

- 哪些频率最强？
- 音频更偏低频还是高频？
- 是否存在明显的基音或谐波结构？

### STFT：时频图把时间带回分析中

STFT 是“短时傅里叶变换”，对窗口化信号分别求频谱：

$$X(t, f) = \sum_{n} x(n) w(n-t) e^{-j2\pi fn/N}$$

它在每个时间帧上生成频谱，有助于判断：

- 声音在哪个时间点出现了新的频率成分？
- 频率变化是否沿着稳定的轨迹移动？
- 是否存在瞬态事件和持续音？

### ACF 与 PACF：时序依赖性诊断

#### ACF（自相关函数）

ACF 衡量时间序列与自身延迟版本的相似度：

$$\rho_k = \frac{\sum_{t=k+1}^{N}(x_t - \bar{x})(x_{t-k} - \bar{x})}{\sum_{t=1}^{N}(x_t - \bar{x})^2}$$

ACF 可以发现周期性和季节性。若在 lag $k$ 上出现显著峰值，说明序列在 $k$ 个时间步后呈现相似结构。

#### PACF（偏自相关函数）

PACF 去掉了中间延迟的影响，帮助判断 AR 模型阶数：

- 若 PACF 在 lag $p$ 后截断，则适合 AR(p) 模型。
- 若 ACF 缓慢衰减，PACF 显著，则说明存在自回归结构。

### 时间序列分析解决了什么问题

传统音频特征告诉我们“这里有什么”，而时间序列分析告诉我们“这些东西是如何在时间上组织的”。

- FFT 解决“频率分布”问题；
- STFT 解决“频率随时间变化”问题；
- ACF/PACF 解决“时间依赖与周期性”问题；
- 这些分析共同回答“音频是否具有可重复的结构？”

## 波动率分析

### 为什么要看波动率

波动率不是信号本身的强度，而是强度的变化速率和波动程度。两个音频可能有相同的平均能量：

- 一个是稳定的电子音乐；
- 一个是急促的鼓点。

波动率分析正是用来区分这类“稳定 vs 剧烈”的差异。

### 滚动波动率

对每条趋势序列计算滚动标准差：

$$\sigma_t = \text{std}(x_{t-w+1}, ..., x_t)$$

这条序列显示了趋势值的瞬时抖动强度。

### ARCH / GARCH

#### ARCH(1)

$$\sigma_t^2 = \omega + \alpha x_{t-1}^2$$

#### GARCH(1,1)

$$\sigma_t^2 = \omega + \alpha x_{t-1}^2 + \beta \sigma_{t-1}^2$$

GARCH 的价值在于：它不是直接建模音频的振幅，而是建模**波动的条件方差**。

- $\beta$ 代表过去波动率的记忆；
- $\beta + \alpha$ 越接近 1，说明波动率具有高度持久性；
- 这个分析帮助我们理解“波动什么时候容易持续”。

### 波动率分析解决了什么问题

它帮助回答：

- 音频的变化是否具有聚类性？
- 一次大幅变化后，后续是否仍然容易出现大幅变化？
- 音频是否具备“波动稳定性”或“灾难性爆发”特征？

## 可预测性分析

### 预测 vs 可预测性

本项目的定位不是简单做一次预测，而是评估一段音频是否**具备预测价值**。

- 如果预测误差小：说明该特征在当前时序上有规律，可预测性高；
- 如果预测误差大：说明该特征本身接近随机，预测困难。

### 经典模型作为可预测性诊断器

#### ARIMA

ARIMA 通过自回归、差分和移动平均建模线性结构。
它回答：

- 音频趋势是否可以用线性关系描述？
- 是否存在稳定的季节性或趋势？

#### HMM

HMM 不是直接预测值，而是寻找隐状态：

- 音频是否由几个内在状态组成？
- 状态之间如何切换？

这一分析揭示音频的“分段行为”与“隐含模式”。

#### LSTM

LSTM 拥有记忆机制，适合捕捉非线性和长期依赖。
它回答：

- 过去多少时刻的信息有价值？
- 该音频是否包含可学习的非线性模式？

#### Transformer

Transformer 的注意力机制擅长捕捉多尺度自相似结构。
它回答：

- 音频是否在不同时间尺度上重复模式？
- 是否存在跨段落的全局依赖？

### 可预测性分析解决了什么问题

它告诉我们：

- 这个音频的某个特征是不是“可预测的信号”；
- 哪些特征更稳定，哪些更随机；
- 预测失败是否来自模型不足，还是来自信号本身。

## 双音频结构分析

### 研究对象从一个音频扩展到两个音频

当上传两个音频时，Audio Lab 不仅比较它们的特征值，更比较它们的结构与波动行为。

### 三种相似度

1. **Trend Similarity**
   - 比较 Energy/Brightness/Complexity/Rhythm 的曲线形状；
   - 强调“趋势是否同步变化”。

2. **Volatility Similarity**
   - 比较波动率层的振幅和聚类特征；
   - 强调“变化的剧烈程度是否一致”。

3. **Structural Similarity**
   - 比较分段结果和隐状态序列；
   - 强调“片段是否在相似位置出现”。

### 双音频分析解决了什么问题

它帮助回答：

- 两段音频是否在节奏与能量上具有相似结构？
- 一段音频的高潮是否对应另一段的高潮？
- 两段音频是否经历了类似的波动过程？

## 系统实现

Audio Lab 的实现不是简单的特征展示平台，而是一个**从信号到时间序列，再到结构化分析**的流水线。其核心流程如下：

1. **音频读取与重采样**
   - 加载音频；统一采样率；单声道处理；归一化。
2. **时间序列构造**
   - 生成 Energy / Brightness / Complexity / Rhythm 四条时间序列。
3. **趋势与波动分析**
   - 计算滑动窗口趋势；生成结构段落；拟合波动率模型。
4. **经典时间序列诊断**
   - 进行 FFT、STFT、ACF、PACF、周期性分析。
5. **可预测性评估**
   - 用 ARIMA、HMM、LSTM、Transformer 评估预测能力。
6. **双音频结构比较**
   - 计算趋势相似度、波动相似度和结构相似度。
7. **结果呈现**
   - 将所有图表与报告组织为 Web 页面与 JSON 输出。
