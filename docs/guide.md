# Audio Lab 使用指南

## Web 界面操作

### 1. 上传音频

访问 `http://localhost:5000`，在首页上传音频文件：

- **Audio 1**（必填）：主音频文件，支持 WAV / MP3 / FLAC 格式
- **Audio 2**（可选）：第二个音频文件，用于双音频对比分析

如果没有上传文件，系统会自动生成一段 5 秒的合成测试音频。

### 2. 选择分析模块

勾选需要运行的分析模块。默认勾选大部分模块（除双音频对比外），可根据需要增删：

| 选项 | 耗时 | 说明 |
|------|------|------|
| 特征提取 | ⚡ 快 | 基础特征，建议始终勾选 |
| 动态趋势分析 | ⚡ 快 | 提取 4 条动态趋势曲线 |
| 音频动态分析 | 🐢 中等 | 趋势 + 波动率 + GARCH + 预测 |
| 模型结构分析 | 🐢 较慢 | 四种模型的联合诊断 |
| 时序分析 | ⚡ 快 | ACF/PACF/周期性/复杂度 |
| 无监督模式发现 | 🐢 中等 | 聚类、变点、motif、NMF |
| 机器学习预测 | 🐢 较慢 | 四模型预测对比（含深度学习） |
| 频带分析 | 🐢 中等 | 低/中/高频段预测评估 |
| 双音频对比 | 🐢 中等 | 需上传第二个音频 |
| 可视化 | ⚡ 快 | 生成所有图表 |

### 3. 参数设置

- **Forecast Horizon**（预测步数）：默认 20，即预测未来的窗口数
- **Mel Bands**（Mel 频带数）：默认 128，影响频谱分辨率

### 4. 查看结果

提交后页面跳转到实时进度页面，通过 SSE（Server-Sent Events）流式推送分析日志和进度条。

分析完成后自动跳转到结果页面，包含：
- 音频基本信息
- 各项分析结果表格
- 可视化图表（PNG 图片）
- 分析日志

## 命令行模式

```bash
# 全部分析（合成音频）
python main.py

# 指定文件
python main.py --audio1 my_song.wav

# 双音频对比
python main.py --audio1 a.wav --audio2 b.wav

# 只运行指定分析
python main.py --audio1 audio.wav --analysis features,dynamics,model_analysis

# 调整预测步数
python main.py --audio1 audio.wav --forecast-horizon 30

# 不保存图片
python main.py --no-save
```

## 各分析模块详解

### 特征提取 (Features)

**做什么**：将原始音频信号转换为不同的数学表示。

**输出内容**：
- **波形 (Waveform)**：时域振幅随时间变化
- **FFT 频谱**：整个音频的平均频率分布
- **STFT 频谱图**：频率随时间的变化热力图
- **Mel 频谱图**：模拟人耳感知的频率-时间表示
- **MFCC**：Mel 频率倒谱系数，常用于语音/音乐识别

**适用场景**：始终运行，是后续分析的基础。

### 动态趋势分析 (Dynamics)

**做什么**：用滑动窗口扫描音频，每个窗口计算 4 个指标，得到 4 条时间曲线。

**四条趋势**：
| 趋势 | 计算方式 | 物理含义 |
|------|---------|---------|
| Energy（能量） | RMS 均方根 | 响度 |
| Brightness（亮度） | 频谱质心 | 声音的明暗 |
| Complexity（复杂度） | 频谱熵 | 声音的"纯净度" |
| Rhythm（节奏） | 起始强度 | 节奏密度 |

还会自动检测结构段落：高潮段(Climax)、安静段(Calm)、积累段(Buildup)、释放段(Release)、过渡段(Transition)。

### 音频动态分析 (Dynamics Analysis)

**做什么**：在趋势层之上增加波动率层。

**Trend Layer**：同动态趋势分析。
**Volatility Layer**：计算每条趋势的滚动波动率 + 拟合 GARCH(1,1) 模型。
**趋势/波动率预测**：用 ARIMA + HMM 对 4 条趋势和 4 条波动率序列进行预测。

**GARCH 模型解读**：
- **α+β（持久性）**：越接近 1，波动率越持久（"一旦开始波动就停不下来"）
- **Half-life**：波动率冲击衰减一半所需的窗口数
- **converged**：GARCH 模型是否成功收敛

### 模型结构分析 (Model Analysis)

**做什么**：四个模型作为"结构侦探"，从不同角度揭示音频的时间结构。

| 模型侦探 | 它回答的问题 |
|---------|------------|
| ARIMA | 每个维度的变化有线性规律吗？ |
| HMM | 音频可以分成几种内在状态？状态之间如何切换？ |
| LSTM | 预测这个音频需要多长的记忆？哪个维度最容易学习？ |
| Transformer | 音频包含哪些时间尺度的自相似结构？ |

**综合概述**：四位侦探的发现会被综合成一段人类可读的报告。

### 时序分析 (Time Series Analysis)

**做什么**：经典的统计诊断。

- **ACF（自相关）**：信号与自己延迟版本的相似度，找周期性
- **PACF（偏自相关）**：去除中间影响后的直接相关性，帮助确定 AR 模型阶数
- **周期性分析**：找到主导频率和周期长度
- **复杂度分析**：过零率 + 样本熵（衡量信号的不可预测性）
- **频谱平坦度**：0=纯音调，1=纯噪声

### 无监督模式发现 (Unsupervised)

**做什么**：不需要任何标签，自动发现音频的内在结构。

- **变点检测**：找到音频"风格变化"的时间点
- **Motif 发现**：找到重复出现的模式片段
- **NMF 频谱分解**：将频谱分解为几个"频谱模板"（类似音频的"颜色成分"）
- **RQA 递归量化**：从相空间重构角度量化系统的确定性/随机性
- **异常检测**：找到最"不寻常"的时刻
- **聚类**：将相似的窗口归为一类

### 机器学习预测 (Prediction)

**做什么**：用四种模型对 Mel 频谱均值进行预测，对比效果。

| 模型 | 类型 | 特点 |
|------|------|------|
| ARIMA | 统计 | 经典，适合线性模式 |
| HMM | 概率 | 学习隐状态，适合分段数据 |
| LSTM | 深度学习 | 循环神经网络，GPU加速 |
| Transformer | 深度学习 | 注意力机制，捕捉长程依赖 |

**输出**：每个模型的 RMSE（均方根误差）和 MAE（平均绝对误差），以及预测对比图。

### 频带分析 (Band Analysis)

**做什么**：将 Mel 频谱分成低频、中频、高频三个频带，分别运行全部预测模型。

- **低频**：贝斯、鼓点、低频节奏
- **中频**：人声、旋律、主要乐器
- **高频**：细节、泛音、空气感

**输出**：每个频带的最佳模型和预测误差排名。

### 双音频对比 (Comparison)

**做什么**：上传两个音频后的综合分析。

**包含**：
1. **音频探索发现**：每个音频的角色画像 + 跨音频段落映射 + 差异对比
2. **动态趋势相似度**：四条趋势的结构相似度（0-100%）
3. **波动率相似度**：四条波动率的结构相似度（0-100%）

**解读**：
- 高相似度（>70%）：两段音频在对应维度上高度一致
- 中等相似度（40-70%）：有共同特征但也有明显差异
- 低相似度（<40%）：两段音频在该维度上迥然不同

## 常见问题

**Q: 分析太慢了怎么办？**
A: ① 取消勾选"模型结构分析"和"机器学习预测"（这两个包含深度学习训练）。② 上传较短的音频。③ 使用 GPU（LSTM/Transformer 会自动使用 CUDA）。

**Q: HMM 显示 NaN 怎么办？**
A: HMM 对数据长度敏感。如果音频太短（<3秒），HMM 可能无法收敛，会自动回退到持久性预测。

**Q: 为什么有些预测值都一样？**
A: 当模型坍缩为单状态时（HMM 常见于短序列），会输出常量预测。这是模型的诚实行为——它表示"我找不到足够的变化规律来做有意义的预测"。

**Q: 支持哪些音频格式？**
A: WAV、MP3、FLAC。依赖 soundfile（WAV/FLAC）和 librosa（MP3）。
