# 使用指南

本指南详细介绍 Audio Lab 平台的使用方法，包括 Web 界面操作和命令行两种方式。

---

## 一、快速开始

### 1.1 环境准备

**系统要求：**
- Python 3.8 或更高版本
- 操作系统：Windows / macOS / Linux

**依赖安装：**
```bash
pip install numpy scipy librosa matplotlib torch scikit-learn hmmlearn
pip install Flask  # Web界面需要
```

### 1.2 启动方式

**方式一：Web 界面（推荐）**
```bash
python app.py
```
然后在浏览器打开 `http://localhost:5000`

**方式二：命令行**
```bash
python main.py --audio1 your_audio.wav
```

---

## 二、Web 界面操作

### 2.1 主界面布局

Web 界面采用现代化的单页应用（SPA）设计，主要分为以下几个区域：

| 区域 | 功能 |
|------|------|
| **顶部导航** | 新建分析、历史记录、文档入口 |
| **参数配置区** | 设置音频文件、分析选项、预测参数 |
| **实时日志区** | 显示分析进度和输出信息 |
| **结果展示区** | 分析完成后显示图表和统计数据 |

### 2.2 上传音频文件

**支持的格式：**
- WAV（推荐，无损）
- MP3
- FLAC
- OGG

**上传方式：**
1. 点击"选择音频文件"按钮
2. 选择本地音频文件
3. 系统自动重采样至目标采样率（默认 16000 Hz）

### 2.3 分析选项配置

#### 基础分析选项

| 选项 | 说明 | 计算内容 |
|------|------|----------|
| **特征提取** | 提取音频基础特征 | 波形、FFT频谱、STFT语谱图、Mel语谱图、MFCC |
| **时间序列分析** | 时序统计分析 | ACF、PACF、周期性、复杂度、白噪声检验 |
| **动态分析** | 音频动态特征 | 能量包络、亮度、复杂度、节奏强度 |
| **无监督学习** | 模式发现 | PCA降维、K-means聚类、异常检测 |

#### 高级分析选项

| 选项 | 说明 | 计算内容 |
|------|------|----------|
| **预测分析** | 时序预测 | ARIMA、HMM、LSTM、Transformer 四模型对比 |
| **频带分析** | 分频带预测 | 低/中/高频带分别建模预测 |
| **动态分析增强** | 波动率建模 | GARCH(1,1)模型、趋势预测 |
| **模型结构分析** | 模型诊断 | 四模型结构一致性检验 |

### 2.4 参数设置

**采样率（Sample Rate）：**
- 默认：16000 Hz
- 建议：语音分析用 16000 Hz，音乐分析用 22050 或 44100 Hz
- 注意：高采样率会增加计算时间

**预测步长（Forecast Horizon）：**
- 默认：20 步
- 范围：1-100 步
- 说明：预测未来多少个时间点的值

**实验名称：**
- 可选参数
- 用于在历史记录中标识本次分析
- 方便后续查找和对比

### 2.5 实时日志

分析过程中，Web 界面会实时显示终端风格的日志输出：

```
[2024-01-15 10:30:45] ===== PHASE 1: FEATURE EXTRACTION =====
[2024-01-15 10:30:45] [1.1] Extracting waveform...
[2024-01-15 10:30:46] [1.2] Computing FFT...
[2024-01-15 10:30:46] [1.3] Computing STFT...
...
[2024-01-15 10:30:52] ===== ANALYSIS COMPLETE =====
```

日志颜色编码：
- **橙色**：阶段标题
- **绿色**：成功完成
- **红色**：错误信息
- **黄色**：警告提示

### 2.6 结果页面

分析完成后自动跳转到结果页面，包含：

1. **概览统计**：音频时长、采样率、预测步长
2. **时间序列分析结果**：周期性、复杂度、白噪声检验
3. **可视化图表**：所有生成的图表
4. **预测结果**：各模型预测对比、误差指标
5. **频带分析**：各频带可预测性排名
6. **分析日志**：完整的终端输出记录

## 三、命令行使用

### 3.1 基本命令

```bash
# 使用合成音频（测试用）
python main.py

# 分析单个音频文件
python main.py --audio1 song.wav

# 分析两个音频文件（对比模式）
python main.py --audio1 song1.wav --audio2 song2.wav

# 指定输出目录
python main.py --audio1 song.wav --output ./my_results

# 指定采样率
python main.py --audio1 song.wav --sr 22050
```

### 3.2 选择分析模块

使用 `--analysis` 参数指定要执行的分析：

```bash
# 只执行特征提取和时间序列分析
python main.py --audio1 song.wav --analysis features,timeseries

# 执行所有分析
python main.py --audio1 song.wav --analysis features,timeseries,dynamics,prediction,band
```

**可用选项：**

| 选项 | 说明 |
|------|------|
| `features` | 特征提取 |
| `timeseries` | 时间序列分析 |
| `dynamics` | 动态分析 |
| `dynamics_analysis` | 动态分析增强（含GARCH） |
| `model_analysis` | 模型结构分析 |
| `unsupervised` | 无监督学习 |
| `prediction` | 预测分析 |
| `band` | 频带分析 |
| `comparison` | 双音频对比 |
| `visualization` | 生成可视化 |

### 3.3 完整参数列表

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `--audio1` | str | None | 第一个音频文件路径 |
| `--audio2` | str | None | 第二个音频文件路径（对比模式） |
| `--output` | str | ./outputs | 输出目录 |
| `--sr` | int | 16000 | 目标采样率 |
| `--duration` | float | 3.0 | 合成音频时长（秒） |
| `--no-save` | flag | False | 不保存图片 |
| `--analysis` | str | None | 指定分析模块 |

## 四、双音频对比模式

当提供两个音频文件时，系统会执行对比分析：

### 4.1 对比内容

- **DTW距离**：动态时间规整距离，衡量两音频相似度
- **动态特征相似度**：能量、亮度、复杂度、节奏的相似度
- **波动率相似度**：GARCH建模后的波动特性对比
- **发现报告**：多维度对比分析报告

### 4.2 使用场景

- 版本对比：同一音频的不同处理版本
- 风格对比：不同风格音频的特性差异
- 质量检测：原始音频与处理后音频的对比

## 五、实验管理

### 5.1 保存实验

在结果页面，可以为当前实验命名并保存：

1. 在左侧边栏的"已保存实验"区域
2. 输入实验名称
3. 点击保存按钮

### 5.2 加载实验

- 在主页点击"历史记录"按钮
- 切换到"已保存实验"标签页
- 点击要加载的实验

### 5.3 实验数据存储

实验数据存储在以下位置：
- 分析结果：`./outputs/<task_id>/`
- 实验索引：`./experiments.json`
- 任务信息：`./outputs/<task_id>/task_info.json`

## 六、最佳实践

### 6.1 音频文件选择

- **时长**：建议 3-30 秒，过长会增加计算时间
- **格式**：优先使用 WAV 格式，避免压缩损失
- **质量**：选择信噪比高的音频，噪声会影响分析结果

### 6.2 参数调优建议

| 场景 | 采样率 | 预测步长 | 分析选项 |
|------|--------|----------|----------|
| 语音分析 | 16000 Hz | 10-20 | features, timeseries, prediction |
| 音乐分析 | 22050 Hz | 20-50 | features, dynamics, band |
| 快速测试 | 8000 Hz | 10 | features, timeseries |
| 深度分析 | 44100 Hz | 30-50 | 全部选项 |

### 6.3 结果解读建议

1. **白噪声检验**：如果通过白噪声检验，说明信号随机性强，预测难度大
2. **频带分析**：关注可预测性排名，选择最可预测的频带进行重点分析
3. **模型对比**：综合 RMSE、MAE、MSE 三个指标选择最佳模型

## 七、故障排除

### 7.1 常见错误

| 错误信息 | 原因 | 解决方法 |
|----------|------|----------|
| `FileNotFoundError` | 音频文件不存在 | 检查文件路径 |
| `LibrosaError` | 音频格式不支持 | 转换为 WAV 格式 |
| `CUDA out of memory` | GPU内存不足 | 减小 batch_size 或使用 CPU |
| `Timeout` | 分析超时 | 减小音频时长或采样率 |

### 7.2 性能优化

- 使用 GPU：安装 PyTorch GPU 版本
- 减少分析选项：只选择需要的分析模块
- 降低采样率：在不影响分析精度的前提下降低采样率
