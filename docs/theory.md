## 1. 声音的物理本质

### 振动产生声音

声音的本质是**介质的机械振动**。当一个物体（如扬声器振膜、声带、琴弦）振动时，它会推动周围的空气分子，产生交替的**压缩**和**稀疏**区域——这就是**声波**。

声波以约 340 m/s 的速度在空气中传播。当这些压力变化到达人耳的鼓膜时，鼓膜随之振动，内耳的毛细胞将振动转换为神经信号，大脑将其解释为声音。

### 三个基本属性

| 属性 | 物理量 | 感知 |
|------|--------|------|
| **振幅 (Amplitude)** | 压力变化的大小 | 响度（大声/小声） |
| **频率 (Frequency)** | 每秒振动次数 (Hz) | 音高（高音/低音） |
| **相位 (Phase)** | 振动的起始位置 | 立体声定位感 |

### 简单例子：纯音

一个 440 Hz 的纯音（钢琴中央 A）在数学上表示为：

$$x(t) = A \cdot \sin(2\pi \cdot 440 \cdot t + \phi)$$

其中：
- $A$ = 振幅（决定音量）
- $440$ = 频率（决定音高）
- $\phi$ = 相位（决定波形起点）
- $t$ = 时间（秒）

现实中的声音远非简单的正弦波——它们由**多种频率成分叠加**而成，这正是我们需要傅里叶变换的原因。

## 2. 从模拟到数字：采样与量化

### 采样定理

计算机只能处理离散的数字。**采样**是将连续的模拟信号转换为离散的数字序列的过程。

每秒采样的次数称为**采样率 (Sample Rate)**，单位是 Hz。常见的 CD 音质采样率是 44100 Hz（每秒 44100 个样本）。

**奈奎斯特-香农采样定理**：要完美重建一个信号，采样率必须**至少是信号最高频率的两倍**。

$$f_s \geq 2 \cdot f_{max}$$

例如：人耳能听到的最高频率约 20000 Hz，因此 CD 采样率 44100 Hz > 2 × 20000 = 40000 Hz ✓。

### 量化

每个样本用多少位来存储称为**位深 (Bit Depth)**。常见的是 16-bit（CD 音质）或 24-bit（专业录音）。16-bit 意味着每个样本有 $2^{16} = 65536$ 个可能的取值。

### 本项目的处理

音频加载后统一**重采样到 16000 Hz 单声道**：
- 16000 Hz 足够覆盖人类语音和大多数音乐分析需求
- 单声道简化处理（立体声取均值）
- 大幅减少计算量

```python
y, sr = loader.load_audio("song.wav", target_sr=16000)
# y = [0.001, 0.003, 0.002, ...]  (80000 个样本 = 5 秒 × 16000)
```

## 3. 时域分析：波形

### 什么是波形

波形是最直观的表示——**振幅随时间的变化**。横轴是时间（秒），纵轴是振幅（归一化到 [-1, 1]）。

### 从波形能看出什么

- **振幅范围**：声音的总体响度
- **零交叉率**：信号穿过零点的频率，粗略反映频率高低
- **包络**：振幅的外轮廓，反映起音/衰减特征
- **瞬态**：突然的尖峰（如打击乐器的起始）

### 代码实现

```python
t, y = features.compute_waveform(y, sr)
# t = [0.0000, 0.0000625, 0.000125, ...]  # 时间轴
# y = [0.001, 0.003, 0.002, ...]          # 振幅值
```

### 局限性

波形不直接告诉我们频率信息。一段波形的"形状"是人眼很难直接解读的——我们无法从波形图中看出"这个声音包含 440 Hz 的成分"。

## 4. 频域分析：傅里叶变换 (FFT)

### 核心思想

**任何周期信号都可以分解为一系列不同频率、振幅和相位的正弦波的叠加。**

这就是傅里叶的核心洞见。FFT（快速傅里叶变换）是实现这一分解的高效算法。

### 数学直觉

傅里叶变换可以理解为"相关性检测"：将信号与每个可能频率的正弦波做内积（点乘），结果的大小表示信号中包含该频率成分的强度。

$$X(k) = \sum_{n=0}^{N-1} x(n) \cdot e^{-j2\pi kn/N}$$

### FFT 输出

```python
freqs, mag = features.compute_fft(y, sr)
# freqs = [0, 1, 2, ..., 7999]    # 频率轴 (0 到 Nyquist = sr/2)
# mag = [0.01, 0.05, 0.02, ...]   # 每个频率的振幅强度
```

### 频谱图解读

- **尖峰 (Peaks)**：该频率成分强 → 音调明显
- **平坦 (Flat)**：各频率均匀 → 噪声特征
- **低频强**：声音"厚重、温暖"
- **高频强**：声音"明亮、尖锐"

### 频谱质心 (Spectral Centroid)

频谱的"重心"——加权平均频率：

$$\text{Centroid} = \frac{\sum f_k \cdot m_k}{\sum m_k}$$

- 高频 → 声音"亮"（如镲片、女高音）
- 低频 → 声音"暗"（如大提琴、男低音）

### FFT 的局限性

FFT 对整个信号计算一个频谱，**丢失了时间信息**。一段音乐中，钢琴声和人声可能出现在不同时间，但 FFT 将它们混在一起。

## 5. 时频分析：短时傅里叶变换 (STFT)

### 核心思想

将信号切成许多小段（窗口），对每一段分别做 FFT。这样我们就能看到**频率如何随时间变化**。

### 三个关键参数

| 参数 | 默认值 | 作用 |
|------|--------|------|
| **窗口大小 (Window Size)** | 2048 样本 | FFT 分辨率。越大 → 频率分辨率越高，时间分辨率越低 |
| **跳步 (Hop Size)** | 512 样本 | 窗口之间移动的距离。越小 → 时间轴越密集 |
| **窗函数 (Window)** | Hann 窗 | 减少频谱泄漏 |

### 海森堡不确定性原理

时间分辨率和频率分辨率之间存在权衡：
- 大窗口 → 好的频率分辨率，差的时间分辨率
- 小窗口 → 好的时间分辨率，差的频率分辨率

### STFT 输出

```python
f_stft, t_stft, spec = features.compute_stft(y, sr)
# f_stft = [0, 7.8, 15.6, ...]           # 频率轴 (1025 个 bin)
# t_stft = [0.0, 0.032, 0.064, ...]      # 时间帧
# spec = [[...], [...], ...]              # (1025, N_frames) 矩阵
```

### 频谱图 (Spectrogram)

频谱图是 STFT 的可视化：横轴=时间，纵轴=频率，颜色=振幅强度。这是音频分析的核心表示。

## 6. 感知频率尺度：Mel 频谱

### 问题：人耳的非线性感知

人耳对频率的感知是**非线性的**：
- 100 Hz → 200 Hz：听起来"翻了一倍"（八度）
- 1000 Hz → 2000 Hz：同样听起来"翻了一倍"
- 但 1000→2000 的线性间距是 100→200 的 10 倍！

这意味着：**线性频率尺度不符合人类听觉感知**。

### Mel 尺度

Mel 尺度是一个**感知频率尺度**，它模拟了人耳对音高的感知：

$$mel = 2595 \cdot \log_{10}\left(1 + \frac{f}{700}\right)$$

- 低频区域：线性近似（人的感知更精细）
- 高频区域：对数压缩（人的感知更粗糙）

### Mel 滤波器组

将线性频谱通过一系列**三角滤波器**转换到 Mel 尺度：

1. 在 Mel 尺度上均匀放置 N 个三角滤波器
2. 每个滤波器覆盖相邻的范围，且与相邻滤波器重叠 50%
3. 低频滤波器窄（分辨率高），高频滤波器宽（分辨率低）

### Mel 频谱输出

```python
mel_freqs, mel_times, mel_spec = features.compute_mel_spectrogram(y, sr, n_mels=128)
# mel_spec: (128, N_frames)  # 128 个 Mel 频带 × 时间帧
```

Mel 频谱更接近"人耳听到的声音"。它是大多数音频分类和识别任务的标准输入。

## 7. 倒谱分析：MFCC

### 什么是倒谱 (Cepstrum)

倒谱是"频谱的频谱"——对频谱再做一次傅里叶变换。它可以将**声源**（声带、乐器振动）和**滤波器**（声道形状、乐器共鸣腔）分开。

### MFCC (Mel-Frequency Cepstral Coefficients)

MFCC 是 Mel 频谱的**离散余弦变换 (DCT)**：

```
原始音频
  → FFT
  → Mel 滤波器组
  → log() 取对数
  → DCT (离散余弦变换)
  → MFCC 系数
```

### 为什么取 log？

- 人耳对响度的感知也是对数式的（分贝尺度）
- 取 log 后将**乘法关系变为加法关系**（声源 × 滤波器 → 声源 + 滤波器）
- 使倒谱分解有效

### 为什么用 DCT？

DCT 能够**去相关**（decorrelate）Mel 频带之间的信息。前几个 MFCC 系数（通常取 13-20 个）捕获了频谱的**包络形状**（音色信息），而高阶系数捕获越来越细节的变化（通常被丢弃）。

### MFCC 系数的含义

| 系数 | 大致对应 |
|------|---------|
| MFCC-0 | 总体能量 |
| MFCC-1 | 频谱的"倾斜度"（亮/暗） |
| MFCC-2-3 | 频谱的"形状"细节 |
| MFCC-4+ | 越来越细节的音色特征 |

### 代码实现

```python
mfcc, times = features.compute_mfcc(y, sr, n_mfcc=13)
# mfcc: (13, N_frames)  # 13 个 MFCC 系数 × 时间帧
```

MFCC 广泛用于语音识别、说话人识别、音乐风格分类等任务。

## 8. 总结：特征管线的逻辑

整个特征提取管线的逻辑链条：

```
声音振动 (物理世界)
    │
    ▼  采样 16000Hz
数字波形 (时域)
    │
    ▼  FFT
频谱 (频域) ─── 看出有哪些频率成分
    │
    ▼  滑动窗口 + FFT
STFT 频谱图 (时频域) ─── 看出频率如何随时间变化
    │
    ▼  Mel 滤波器组（模拟人耳）
Mel 频谱 ─── 更接近人类听觉的表示
    │
    ▼  log + DCT
MFCC ─── 压缩的音色特征
```

每一步都解决了上一步的某个局限性：
1. **波形** → 缺少频率信息 → **FFT**
2. **FFT** → 缺少时间信息 → **STFT**
3. **STFT** → 线性频率不符合人耳 → **Mel 尺度**
4. **Mel 频谱** → 频带相关、维度高 → **MFCC**

在我们项目的代码中，所有四个表示都被计算和保存，因为它们在不同的分析中有不同的用途：
- **波形**：用于动态趋势提取（RMS 能量、过零率等）
- **FFT**：用于频谱平坦度、频谱质心
- **Mel 频谱**：用于机器学习预测、频带分析
- **MFCC**：用于相似度分析、模式发现
