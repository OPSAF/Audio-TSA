## 1. 动态趋势层

**模块文件**: `audiots/dynamics.py`
**核心函数**: `extract_dynamics()`

### 算法原理

用长度为 0.5s、步长为 0.25s 的滑动窗口扫描音频。对每个窗口执行 FFT 并计算四个标量指标，得到四条等长的时间序列曲线。

### 四个趋势的数学定义

| 趋势 | 公式 | 范围 | 物理含义 |
|------|------|------|---------|
| **Energy** | $E = \sqrt{\frac{1}{N}\sum_{i=1}^{N} x_i^2}$ | [0, 1] | RMS 能量 = 响度 |
| **Brightness** | $C = \frac{\sum f_k \cdot m_k}{\sum m_k}$ | [0, sr/2] | 频谱质心 = 音色明暗 |
| **Complexity** | $H = -\sum p_k \log_2(p_k), \; p_k = m_k/\sum m_k$ | [0, ~15] | 频谱熵 = 声音"纯净度" |
| **Rhythm** | $R = \frac{1}{N-1}\sum \vert x_{pre}[i]^2 - x_{pre}[i-1]^2\vert$ | [0, ~1] | 频谱通量 = 起始点密度 |

其中 $x_{pre}$ 是预加重后的信号（$x_{pre}[i] = x[i] - 0.97 \cdot x[i-1]$），用于增强高频以利于起始点检测。

### 结构分段检测

`detect_structural_segments()` 基于规则将每个窗口分类：

```
if energy > 85百分位 AND rhythm > 85百分位:  "高潮段 (Climax)"
elif energy < 25百分位 AND rhythm < 25百分位: "安静段 (Calm)"
elif 能量导数 > +1σ:                           "积累段 (Buildup)"
elif 能量导数 < -1σ:                           "释放段 (Release)"
elif complexity > 80百分位:                    "过渡段 (Transition)"
else:                                          "持续段 (Sustained)"
```

### 趋势相似度

`compute_dynamics_similarity()` 比较两条趋势曲线。不是简单比较数值大小，而是比较**形状**：

| 指标 | 权重 | 含义 |
|------|------|------|
| Pearson r | 15% | 线性相关性 |
| Spearman r | 15% | 单调关系（排名相关性） |
| Derivative r | 35% | 形状相似度（导数相关性） |
| Peak overlap | 25% | 峰值位置的重叠度 |
| Distribution EMD | 10% | 分布形状相似度 |

**权重偏重形状**（导数+峰值=60%）而非数值（Pearson+Spearman=30%），因为两段音频即使在不同的响度级别，也可能有相似的结构模式。

## 2. 波动率层

**模块文件**: `audiots/volatility.py`
**核心函数**: `compute_volatility_layer()`

### 为什么需要波动率分析

趋势层告诉我们"能量有多高"。波动率层告诉我们"能量的变化有多剧烈"。

一个持续的嗡嗡声（高能量、低波动率）vs 一段激烈的鼓点（高能量、高波动率）在趋势层可能看起来相似，但在波动率层完全不同。

### 滚动波动率

对每条趋势序列计算滚动标准差：

$$\sigma_t = \text{std}(x_{t-w+1}, x_{t-w+2}, ..., x_t)$$

其中 $w=10$ 为滚动窗口（前 $w-1$ 个值用扩展窗口）。

### GARCH(1,1) 模型

GARCH（广义自回归条件异方差）模型捕捉**波动率聚类**现象——金融市场和音频信号都有的特征："大波动倾向于跟随大波动，小波动倾向于跟随小波动"。

**GARCH(1,1) 方程**：

$$\sigma_t^2 = \omega + \alpha \cdot r_{t-1}^2 + \beta \cdot \sigma_{t-1}^2$$

| 参数 | 含义 |
|------|------|
| $\omega$ | 基础波动率水平 |
| $\alpha$ | 新信息对波动率的影响速度 |
| $\beta$ | 旧波动率的持续性 |
| $\alpha + \beta$ | 持久性（越接近1，波动率越持久） |

**半衰期**: $\frac{\ln(0.5)}{\ln(\alpha+\beta)}$ — 波动率冲击衰减一半所需的窗口数。

### 拟合策略

1. 优先使用 statsmodels 的 `arch` 包
2. 失败则使用纯 scipy 的 MLE（L-BFGS-B 优化，施加平稳性约束 $\alpha+\beta < 1$）
3. 再失败则回退到常量波动率

### 波动率相似度

`compute_volatility_similarity()` 比较两个音频的波动率结构：

| 指标 | 权重 | 含义 |
|------|------|------|
| Pearson r | 30% | 波动率曲线的线性相关 |
| Derivative r | 35% | 波动率曲线的形状相似 |
| Distribution | 20% | 波动率值的分布重叠 |
| GARCH param | 15% | GARCH 参数的相似度 |

## 3. 模型结构分析

**模块文件**: `audiots/model_analysis.py`
**核心函数**: `analyze_model_ensemble()`

### 设计哲学

四个模型不再是"预测比赛选手"，而是**四个不同角度的结构侦探**。每个模型训练后，提取的不是预测值，而是**结构洞察**。

### ARIMA：线性结构侦探

**问题**: 每个维度的变化有线性规律吗？

**方法**:
1. ADF 检验确定差分阶数 $d$
2. 网格搜索 $(p,d,q)$ 空间，AIC 选优
3. Ljung-Box 检验残差是否为白噪声（模型是否充分捕捉了结构）
4. ACF 峰值检测找周期性

**输出的洞察**:
- 最优 ARIMA 阶数
- 趋势类型：均值回复 / 随机游走 / 振荡 / 趋势 / 移动平均 / 白噪声
- "残差是白噪声" = 线性结构已被充分捕捉
- "残差非白噪声" = 存在非线性结构（需要 LSTM/Transformer）

### HMM：隐状态发现侦探

**问题**: 音频可以自动分为几种内在状态？

**方法**:
1. 将四条趋势组合为 $(N_{windows}, 4)$ 特征矩阵
2. 拟合高斯 HMM（5 次随机重启，选最高得分）
3. 预测隐状态序列
4. 自动标注状态（根据各状态的均值特征 vs 全局百分位阈值）

**输出的洞察**:
- 发现了多少种状态
- 每个状态的"画像"（能量/亮度/复杂度/节奏的平均值）
- 状态转移模式（是否循环？）
- 状态区分度（between/within variance 比）
- 段落结构（连续同状态 → 一个段落）

### LSTM：记忆与可学性侦探

**问题**: 预测这个音频需要多长的记忆？哪个维度最可学？

**方法**:
1. 用不同 lookback (10, 20, 30) 训练 LSTM 预测能量趋势
2. 找"肘点"——增加 lookback 不再改善 loss 的点
3. 对四个维度分别训练 LSTM，记录各自的 MSE

**输出的洞察**:
- 最优记忆长度（窗口数 + 秒数）
- 记忆体制：短时 (<1s) / 中等 (1-3s) / 长时 (>3s)
- 各维度可学性排名
- "最可学" vs "最难学"的维度

### Transformer：注意力模式侦探

**问题**: 音频包含哪些时间尺度的自相似结构？

**方法**:
1. 训练 Transformer 预测能量趋势
2. 尝试提取注意力权重（如不可用，回退到 ACF 峰值聚类法）

**输出的洞察**:
- 每个注意力头关注的时间 lag
- 注意力头多样性（多少种不同时间尺度共存）
- 时间尺度的解释：局部 (~0.5s) / 短程 (~2s) / 中程 (~5s) / 长程 (10s+)

### 综合概述

`_compose_ensemble_summary()` 将四位侦探的发现合成为一段中文自然语言的综合报告。

## 4. 时序统计诊断

**模块文件**: `audiots/analysis.py`

### ACF (自相关函数)

$$\rho_k = \frac{\sum_{t=k+1}^{N}(x_t - \bar{x})(x_{t-k} - \bar{x})}{\sum_{t=1}^{N}(x_t - \bar{x})^2}$$

测量信号与自身延迟版本的相似度。$\rho_k$ 高 → 信号有 $k$ 步的周期性。

### PACF (偏自相关函数)

在控制中间时间点影响后，测量 $x_t$ 和 $x_{t-k}$ 的直接相关性。用于确定 AR 模型阶数——PACF 在 lag $p$ 后截断 → AR(p)。

### 周期性分析

`analyze_periodicity()`:
1. 对信号做 FFT
2. 找主导频率（最大振幅的频率 bin）
3. 在 ACF 中验证（找显著峰值）
4. 输出主导周期（秒和样本数）

### 频谱平坦度 (Spectral Flatness)

$$\text{Flatness} = \frac{\sqrt[N]{\prod m_k}}{\frac{1}{N}\sum m_k}$$

几何平均 / 算术平均。0 = 纯音调（频谱有一个尖峰），1 = 纯噪声（频谱完全平坦）。

### 样本熵 (Sample Entropy)

测量信号的**不可预测性**。对长度为 m 的模板，寻找在容差 r 内相似的匹配。计算方式的直觉：

- 低熵 → 模式重复、可预测（如纯音、周期信号）
- 高熵 → 模式不重复、不可预测（如白噪声、混沌信号）

## 5. 无监督模式发现

**模块文件**: `audiots/unsupervised.py`
**核心函数**: `explore_unsupervised()`

### 六阶段分析管线

**阶段 1 — 变点检测 (Change Point Detection)**

用 MMD (Maximum Mean Discrepancy) 代理：比较每个候选点前后的特征分布差异。

$$\text{MMD} \propto \frac{|\mu_{before} - \mu_{after}|^2}{\sigma^2_{pooled}}$$

峰值位置 = 结构变化边界。

**阶段 2 — Motif 发现**

构建子序列距离矩阵（z-score 归一化后的余弦距离），找最近邻对，合并重叠匹配为 motif 组。

**阶段 3 — NMF 频谱分解**

将 Mel 频谱分解为 K 个非负的"频谱模板"（基矩阵 W）和它们的激活时间序列（系数矩阵 H）：

$$V \approx W \cdot H$$

每个模板可以理解为一种"频谱成分"（如低频嗡嗡声、中频人声、高频嘶嘶声）。

**阶段 4 — RQA 递归量化**

用时间延迟嵌入重构相空间轨迹，绘制递归图（Recurrence Plot），计算：
- **Recurrence Rate**: 系统回到相同状态的比例
- **Determinism**: 对角线上递归点的比例（可预测性）
- **Laminarity**: 垂直线段的比例（间歇性/层流状态）

**阶段 5 — 异常检测**

计算每个窗口到特征质心的 Mahalanobis 距离。距离最大的窗口 = 最不寻常的时刻。

**阶段 6 — 聚类**

K-Means 在 z-score 特征上聚类，silhouette score 评估质量。

## 6. 机器学习预测

**模块文件**: `audiots/prediction.py`

### 四种预测模型

**ARIMA(p,d,q)**
$$\left(1 - \sum_{i=1}^{p} \phi_i L^i\right)(1-L)^d X_t = \left(1 + \sum_{i=1}^{q} \theta_i L^i\right)\varepsilon_t$$

- 经典统计时间序列模型
- 适合线性模式
- 自动网格搜索最优 (p,d,q)，多 AIC 候选

**HMM (Gaussian Hidden Markov Model)**

- 学习 $K$ 个隐状态，每个状态对应一个高斯分布
- 预测：从当前最可能状态出发，依转移矩阵选择下一状态，输出该状态的均值
- 对 HMM 退化（单状态坍缩、transmat 行和为 0）有 fallback 保护

**LSTM (Long Short-Term Memory)**

- 2 层：`LSTM(input=1, hidden=64) → FC(64 → horizon)`
- Adam 优化器，AMP 混合精度，梯度裁剪
- 训练前检查缓存避免重复计算
- 数据量不足时自动跳过

**Transformer Encoder**

- `Linear → PositionalEncoding → TransformerEncoder(2 layers, 4 heads) → FC`
- StepLR 学习率调度器
- 注意力机制可以学习远程依赖
- 同样有缓存和数据量检查

### 预测缓存

`PredictionCache` 基于 `(series_hash, model_type, lookback, horizon, epochs)` 的 SHA256 哈希避免同一数据上的重复训练。线程安全，LIFO 驱逐。

### GPU 支持

自动检测 CUDA 设备（如 NVIDIA RTX 3060），将数据和模型移至 GPU 训练，训练完成后清理缓存。

## 7. 频带可预测性分析

**模块文件**: `audiots/band_analysis.py`

### 思路

将 Mel 频谱均分为三个频带，对每个频带的时间均值序列运行全部预测模型。核心问题：**音频的哪个频率范围最"有规律"（最容易预测）？**

### 频带划分

| 频带 | Mel bin 范围 | 大致对应 |
|------|-------------|---------|
| 低频 (Low) | 0 ~ n_mels/3 | 贝斯、底鼓、低频节奏 (<500 Hz) |
| 中频 (Mid) | n_mels/3 ~ 2n_mels/3 | 人声、吉他、键盘 (500-2000 Hz) |
| 高频 (High) | 2n_mels/3 ~ n_mels | 镲片、泛音、空气感 (>2000 Hz) |

### 输出

- 每个频带的最佳预测模型
- 可预测性排名（RMSE 从低到高）
- 错误汇总表格

## 8. 双音频对比发现

**模块文件**: `audiots/discovery.py`
**核心函数**: `explore()`

### 设计哲学

**不做单一相似度分数**。相似度是主观的——两段音频可能在节奏上相似但在音色上不同。本模块**发现并报告**多维度的关系，由用户判断相似性。

### 三项分析

#### 音频性格画像 (Character Profile)

对每个音频自动生成文字描述：
- **节奏特征**: 估算 BPM + 密度分类
- **能量特征**: 趋势方向 + 峰值时间
- **音色特征**: 明暗程度 + 纯净/噪声

#### 跨音频段落映射 (Cross-Discovery)

对每个趋势维度，在音频 A 中找"有趣点"（峰值和谷值），提取上下文窗口的形状（导数），然后在音频 B 中滑动搜索最匹配的窗口。

**置信度**: 60% 导数相关 + 40% 原始 Pearson 相关

#### 差异分析 (Contrast)

对每个维度比较分布：
- 归一化差异比 > 1.0: 显著差异
- 分布重叠度 (Wasserstein EMD 反转)

## 9. 音频相似度分析

**模块文件**: `audiots/similarity.py`
**核心函数**: `analyze_similarity()`

### 六阶段管线

1. **窗口特征提取**: 61 维特征向量（节奏、能量、音色、和声、时序、MFCC统计）
2. **分布匹配**: 逐维度 Wasserstein 距离
3. **局部相似矩阵**: PCA 白化 + 余弦相似 + 高斯核
4. **全局聚合**: 60% 简单匹配 + 25% 熵加权 + 15% 分布相似
5. **模式发现**: 高潮检测、重复分析、新颖度曲线
6. **标注**: 高相似段落的特征标签

### 核心创新

**长度无关、对齐无关**——不依赖 DTW 或时间对齐，而是比较音频的**统计指纹**。两段不同长度的相似音频（如同一首歌的不同段落）也能被正确识别。
