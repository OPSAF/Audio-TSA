# 常见问题 FAQ

本文档收集用户常见问题及解答。

---

## 一、安装与环境

### Q1: 安装依赖时出现错误怎么办？

**A:** 常见原因及解决方法：

| 错误类型 | 原因 | 解决方法 |
|----------|------|----------|
| `pip install` 失败 | 网络问题 | 使用国内镜像源 |
| `librosa` 安装失败 | 缺少音频解码库 | 安装 `ffmpeg` |
| `torch` 安装慢 | 包体积大 | 使用预编译版本 |

**推荐安装命令：**
```bash
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple numpy scipy matplotlib
pip install librosa
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

### Q2: 需要什么版本的 Python？

**A:** 推荐 Python 3.8 - 3.11。Python 3.12 可能存在部分库兼容问题。

### Q3: 需要安装 CUDA 吗？

**A:** 不强制。深度学习模型会自动检测 CUDA：
- 有 CUDA：使用 GPU 加速
- 无 CUDA：使用 CPU 运行

## 二、音频文件

### Q4: 支持哪些音频格式？

**A:** 支持以下格式：
- WAV（推荐）
- MP3
- FLAC
- OGG
- M4A

### Q5: 音频文件大小有限制吗？

**A:** 无硬性限制，但建议：
- 时长：3-30 秒（过长会增加计算时间）
- 采样率：8000-48000 Hz
- 单声道优先（立体声会被自动转为单声道）

### Q6: 为什么我的音频被重采样了？

**A:** 系统会将音频重采样至目标采样率（默认 16000 Hz）以统一处理。可在参数设置中修改目标采样率。

### Q7: 音频文件路径包含中文会出错吗？

**A:** 不会。系统支持中文路径。

## 三、分析选项

### Q8: 各分析选项的计算时间大概是多少？

**A:** 以 10 秒音频、16000 Hz 采样率为例：

| 分析选项 | 大致时间 |
|----------|----------|
| 特征提取 | 1-2 秒 |
| 时间序列分析 | 1-2 秒 |
| 动态分析 | 2-3 秒 |
| 预测分析（四模型） | 10-30 秒 |
| 频带分析 | 30-60 秒 |
| 全部分析 | 1-2 分钟 |

### Q9: 可以只运行部分分析吗？

**A:** 可以。在 Web 界面勾选需要的分析选项，或命令行使用 `--analysis` 参数指定。

### Q10: 预测步长应该设多少？

**A:** 建议：
- 快速测试：10 步
- 常规分析：20 步（默认）
- 长期预测：30-50 步

注意：步长越大，预测不确定性越高。

## 四、模型与预测

### Q11: 四种预测模型各有什么特点？

**A:**

| 模型 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| ARIMA | 快速、可解释 | 仅线性 | 平稳序列 |
| HMM | 状态建模 | 马尔可夫假设 | 状态切换 |
| LSTM | 长期记忆 | 训练慢 | 复杂序列 |
| Transformer | 全局依赖 | 计算量大 | 长序列 |

### Q12: 为什么预测效果不好？

**A:** 可能原因：
1. **信号接近白噪声**：检查白噪声检验结果
2. **数据量不足**：增加音频时长
3. **模型选择不当**：尝试其他模型
4. **参数需要调优**：调整预测步长、窗口大小

### Q13: 深度学习模型训练很慢怎么办？

**A:**
1. 安装 CUDA 版本的 PyTorch
2. 减小批大小（batch_size）
3. 减少训练轮数
4. 使用更简单的模型（ARIMA）

### Q14: 可以保存训练好的模型吗？

**A:** 目前每次分析都会重新训练。后续版本会支持模型保存和加载。

## 五、结果解读

### Q15: 白噪声检验通过了意味着什么？

**A:** 意味着信号随机性强，难以预测。建议：
- 关注频带分析，寻找可预测的频带
- 降低预测期望
- 考虑其他分析角度（如特征提取）

### Q16: RMSE、MAE、MSE 应该看哪个？

**A:**
- **RMSE**：最重要，对大误差敏感
- **MAE**：辅助参考，对异常值不敏感
- **MSE**：优化目标，不直接解读

综合比较，选择 RMSE 最小的模型。

### Q17: 为什么不同频带的预测效果差异很大？

**A:** 这是正常的：
- 低频带：通常规律性强，易于预测
- 高频带：通常随机性强，难以预测

这反映了音频信号的物理特性。

### Q18: GARCH 参数怎么解读？

**A:**
- **α + β 接近 1**：波动持久，波动聚集现象明显
- **α + β 远小于 1**：波动快速衰减
- **ω**：基准波动水平

## 六、Web 界面

### Q19: Web 界面打不开怎么办？

**A:**
1. 确认 `app.py` 正在运行
2. 检查端口 5000 是否被占用
3. 尝试访问 `http://127.0.0.1:5000`

### Q20: 上传文件后没有反应？

**A:**
1. 检查文件格式是否支持
2. 检查文件大小是否过大
3. 查看浏览器控制台是否有错误

### Q21: 分析卡住了怎么办？

**A:**
1. 查看终端日志
2. 刷新页面重试
3. 减少分析选项

### Q22: 如何保存分析结果？

**A:**
- Web 界面：分析结果自动保存在 `./outputs/` 目录
- 可以在结果页面为实验命名保存
- 历史记录可在主页"历史记录"中查看

## 七、命令行

### Q23: 命令行参数怎么用？

**A:** 示例：
```bash
# 基本用法
python main.py --audio1 song.wav

# 指定采样率
python main.py --audio1 song.wav --sr 22050

# 选择分析模块
python main.py --audio1 song.wav --analysis features,timeseries,prediction

# 双音频对比
python main.py --audio1 a.wav --audio2 b.wav
```

### Q24: 如何查看命令行帮助？

**A:** 运行：
```bash
python main.py --help
```

## 八、性能优化

### Q25: 分析速度太慢怎么办？

**A:**
1. 减少分析选项
2. 降低采样率
3. 缩短音频时长
4. 使用 GPU
5. 关闭不需要的可视化（`--no-save`）

### Q26: 内存不足怎么办？

**A:**
1. 减小批大小
2. 缩短音频时长
3. 降低采样率
4. 关闭其他程序

### Q27: GPU 利用率低怎么办？

**A:** 深度学习模型默认参数较小，GPU 利用率可能不高。可以：
1. 增加批大小
2. 增加模型层数
3. 增加隐藏单元数

## 九、故障排除

### Q28: 出现 "CUDA out of memory" 错误？

**A:**
1. 减小批大小
2. 减小模型大小
3. 使用 CPU 运行

### Q29: 出现 "module not found" 错误？

**A:** 安装缺失的依赖：
```bash
pip install <module_name>
```

### Q30: 结果与预期不符？

**A:**
1. 检查输入音频是否正确
2. 检查参数设置
3. 查看完整日志
4. 尝试默认参数
