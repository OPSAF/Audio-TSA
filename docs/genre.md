## 一、项目概述

### 1.1 GTZAN 数据集

**GTZAN** 是音乐信息检索领域最著名的数据集之一，由 G. Tzanetakis 和 P. Cook 于 2002 年发布。

**数据集统计：**

| 属性 | 值 |
|------|-----|
| 总样本数 | 1000 首 |
| 音乐风格 | 10 类 |
| 每类样本数 | 100 首 |
| 音频格式 | WAV |
| 采样率 | 22050 Hz |
| 时长 | 30 秒/首 |

**10 种音乐风格：**

| 编号 | 风格 | 英文名 | 特点 |
|------|------|--------|------|
| 1 | 蓝调 | Blues | 12小节结构，忧郁情感 |
| 2 | 古典 | Classical | 交响乐，复杂结构 |
| 3 | 乡村 | Country | 简单和弦，叙事歌词 |
| 4 | 迪斯科 | Disco | 强节奏，舞曲风格 |
| 5 | 嘻哈 | Hip-hop | 说唱，重低音 |
| 6 | 爵士 | Jazz | 即兴，复杂和弦 |
| 7 | 金属 | Metal | 失真吉他，强节奏 |
| 8 | 流行 | Pop | 朗朗上口，简单结构 |
| 9 | 雷鬼 | Reggae | 反拍节奏，放松感 |
| 10 | 摇滚 | Rock | 强节奏，电吉他 |

### 1.2 分析目标

1. **特征分析**：分析各风格的特征差异
2. **可预测性**：评估各风格的时间序列可预测性
3. **分类模型**：构建风格分类模型
4. **混淆分析**：分析易混淆的风格对

## 二、分析方法

### 2.1 特征提取

对每首音频提取以下特征：

| 特征类别 | 具体特征 |
|----------|----------|
| 时域特征 | 能量、零交叉率 |
| 频域特征 | 频谱质心、频谱带宽、频谱平坦度 |
| 时频特征 | Mel语谱图、MFCC |
| 复杂度特征 | 样本熵、近似熵 |
| 动态特征 | 能量包络、节奏强度 |

### 2.2 时间序列分析

对每种风格进行：
- 白噪声检验
- 周期性分析
- ACF/PACF 分析
- 频带可预测性分析

### 2.3 分类模型

计划构建以下分类模型：

| 模型 | 说明 |
|------|------|
| 传统机器学习 | SVM、随机森林、KNN |
| 深度学习 | CNN、LSTM、Transformer |
| 集成方法 | XGBoost、LightGBM |

## 六、思考与讨论

### 6.1 预期挑战

1. **风格边界模糊**：某些风格间界限不清晰
   - Blues 与 Jazz
   - Rock 与 Metal
   - Pop 与 Disco

2. **类内差异大**：同一风格内部差异可能很大
   - Classical 包含巴洛克、浪漫主义等子风格
   - Rock 包含多种子流派

3. **特征选择**：哪些特征对风格区分最重要？

### 6.2 研究问题

1. 哪种音乐风格最容易预测？
2. 哪种风格最难预测？
3. 哪些风格容易混淆？
4. 时间序列可预测性与风格分类的关系？

### 6.3 应用前景

- 音乐推荐系统
- 自动音乐标注
- 音乐检索
- 音乐生成
