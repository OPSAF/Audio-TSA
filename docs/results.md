## 总体原则

### 分析不是"正确答案"而是"提供证据"

Audio Lab 的分析结果**不是**对音频的评判，而是**多角度的证据呈现**。

- **相似度** 不是客观真理——它是特定方法下的估计值
- **模型预测** 不是"标准答案"——高误差可能意味着信号本身就难以预测
- **HMM 状态** 不是"真正的段落"——它是统计规律的体现

### 信号越短，结果越不确定

滑动窗口的窗口数是分析的基本单元数。5 秒的音频在 0.5s 窗口/0.25s 步长下只有约 19 个窗口——这对于统计模型（HMM、GARCH）来说非常有限。深度学习模型对数据量更敏感，太短的序列会被自动跳过。

### 所有数字都应该在上下文中理解

一个 RMSE 是 0.2 还是 2.0，取决于数据的尺度。不要仅仅比较绝对值，要注意：
- 与其他模型在同一数据上的表现对比
- 预测值的变化范围
- 数据本身的噪声水平

## 特征提取图表

### 波形图 (waveform.png)

横轴：时间（秒），纵轴：振幅 [-1, 1]

**看什么**：
- 振幅的包络（轮廓）反映了声音的起音/衰减
- 密集的波形 = 高频成分；稀疏的波形 = 低频成分
- 静音段 = 振幅接近 0
- 瞬态尖峰 = 打击乐器的起始点

### FFT 频谱图 (fft_spectrum.png)

横轴：频率（Hz），纵轴：振幅强度

**看什么**：
- 尖峰位置 = 音调频率（如 440Hz = 钢琴中央 A）
- 尖峰的高度 = 该频率的强度
- 整体趋势：下降快 = 声音偏暗，下降慢 = 声音偏亮
- 没有明显尖峰 = 噪声特征

### Mel 频谱图 (mel_spectrogram.png)

横轴：时间，纵轴：Mel 频率（低→高），颜色：能量强度（亮=强）

**看什么**：
- 水平亮带 = 持续频率成分（如持续的音符）
- 垂直亮线 = 瞬态事件（如打击乐器的起始）
- 低频亮带多 = 贝斯/底鼓的节奏模式
- 中高频变化 = 旋律和和声进行

## 动态趋势与波动率图表

### 趋势图 (dynamics/trends.png)

四条曲线，横轴时间，纵轴归一化 Z-score。

**看什么**：
- 四条曲线的整体走向：一起上升 = 音频在"积累"，一起下降 = 音频在"消退"
- 能量和节奏的峰值是否对齐：通常高潮段两者同时达到峰值
- 复杂度的尖峰：可能表示过渡段（频谱不稳定）
- 背景色带 = 结构段落标签（高潮/安静/积累等）

### 波动率层 (volatility/volatility_layer.png)

趋势曲线叠加波动率半透明带。

**看什么**：
- 半透明带的宽度 = 波动率大小（宽度大 = 该时段变化剧烈）
- 波动率带宽在哪些地方突然增大 = 结构变化点
- 高波动率之后是否持续高波动率 = 波动率聚类现象

### GARCH 诊断 (volatility/garch_energy.png)

**看什么**：
- 条件波动率曲线（由 GARCH 模型估计）vs 实际波动率
- 如果两条曲线贴合好 → GARCH 模型捕捉了波动率动态
- persistence (α+β) 接近 1 → 波动率高度持久
- converged=False → GARCH 未能收敛，波动率可能近似常量

## 模型分析报告

### ARIMA 分析

```
energy: AR(1) 均值回复过程——变化有惯性，倾向于回到均值。
残差为白噪声 (Ljung-Box p=0.342)，模型已充分捕捉结构

complexity: 接近白噪声——该维度的逐窗变化几乎完全随机、无规律可循
```

**解读**：
- "残差为白噪声" = 线性结构已被充分描述，不需要更复杂的模型
- "接近白噪声" = 该维度本质上随机，任何模型都无法可靠预测
- "非平稳" = 该维度有明显趋势，需要先差分才能建模
- "检测到约 X 窗口的周期" = 该维度有规律的周期性波动

### HMM 分析

```
HMM 发现 3 种内在状态: 安静段, 活跃段, 高潮段
转移模式: 状态0→状态1(75%) → 状态1→状态2(90%) → 状态2→状态0(40%)
状态区分度: 82%（状态边界清晰）。结构呈循环性。
```

**解读**：
- 状态数量反映了音频的动态多样性：2 种 = 简单（如纯语音），4+ 种 = 复杂（如复杂编曲的音乐）
- 区分度 > 80%：状态之间有明确界限
- 区分度 < 50%：状态之间有重叠，边界模糊
- "循环性"：状态有规律地循环（如主歌→副歌→主歌）
- "状态倾向于持续"：状态切换不频繁（如一段长的持续音）

### LSTM 分析

```
最优记忆长度: 12 窗口 (~3.0s)。中等记忆（1-3s）。
各维度可学性: energy=0.012, brightness=0.045, rhythm=0.089, complexity=0.210
最容易学习: energy，最难学习: complexity（接近随机）
```

**解读**：
- 短时记忆 (<1s)：音频的局部变化占主导，远距离信息无用
- 中等记忆 (1-3s)：需要约 1-3 秒的上下文
- 长时记忆 (>3s)：音频具有大尺度结构（如古典乐章的起承转合）
- 最难学习的维度如果接近随机 → 该维度的变化没有可学习的模式

### Transformer 分析

```
检测到 3 个时间尺度——多层次结构
Head 0: lag=2 (0.5s) — 局部相邻依赖
Head 1: lag=16 (4.0s) — 长程结构层
Head 2: lag=7 (1.8s) — 短程节奏层
```

**解读**：
- 3 个不同时间尺度 = 音频具有多层次结构（局部节奏 + 短语 + 段落）
- 1 个时间尺度 = 音频结构单一，主要由一个时间尺度的模式驱动
- 0 个时间尺度 = 未检测到明显的跨时间自相似结构（可能是随机信号）

## 预测对比图表

### 预测对比 (prediction_comparison.png)

多条曲线叠加：真实值 + 各模型预测值。

**看什么**：
- 哪条预测线与"真实值"线最接近 = 该模型在本次数据上表现最好
- ARIMA 预测过于平滑 → 只捕捉了线性趋势
- LSTM/Transformer 预测有波动 → 捕捉到了非线性模式
- HMM 预测呈台阶状 → 以状态均值为基础

### 模型误差对比 (model_error_comparison.png)

分组柱状图：每个模型的 RMSE 和 MAE。

**理解误差大小**：
- RMSE 比 MAE 大很多 → 有些预测点偏差特别大（RMSE 对大误差更敏感）
- 所有模型误差相近 → 信号可能接近白噪声，没有可学习的模式
- 深度学习模型明显优于 ARIMA → 信号包含非线性结构

### 频带错误热力图 (band_error_heatmap.png)

热力图：频带 × 模型的 RMSE。

**看什么**：
- 低频 RMSE 低 → 节奏/贝斯部分规律性强
- 高频 RMSE 高 → 细节/泛音部分随机性大
- 某频带所有模型都表现差 → 该频带本质上"无规律"

## 双音频对比图表

### 自发现图 (discovery/self_discovery_a.png)

单个音频的结构肖像。

**看什么**：
- 右上角的角色画像：用文字描述这个音频的"性格"
- 中间的趋势曲线 + 彩色背景：结构段落的分布
- 底部：高潮点（红色虚线）和安静点（蓝色虚线）
- 重复 motif：音频中重复出现的模式片段

### 跨音频段落映射 (discovery/segment_mapping.png)

两个音频的同一维度的趋势曲线叠加，彩色区域和箭头连接对应的片段。

**看什么**：
- 箭头从一个音频的某个区域指向另一个音频的某个区域 → 算法认为这些片段结构上对应
- 箭头的颜色 = 该维度的颜色（红色=能量，蓝色=亮度等）
- 置信度高 (conf > 0.7) → 可靠的对应关系
- 置信度低 (conf < 0.6) → 弱对应，可能偶然

### 发现摘要仪表板 (discovery/discovery_summary.png)

单页汇总。

**看什么**：
- 左上：自然语言概述
- 中上：两个音频的角色卡片
- 下部：关键对比（差异程度+分布重叠）
- 右半：跨音频发现列表（每个发现含匹配数、置信度、预览）

## 常见误解

### ❌ "相似度 60% 说明这俩基本一样"

相似度分数依赖于算法和权重。60% 意味着在多维度统计意义上存在中等强度的关联，不代表"60% 相同"。

### ❌ "HMM 发现的 3 种状态就是主歌/副歌/桥段"

HMM 发现的是**统计聚类**，不是语义标签。状态 1 可能对应副歌，也可能对应"高能量+高节奏密度"的任何段落。

### ❌ "LSTM 误差低于 ARIMA，所以 LSTM 更好"

LSTM 和 ARIMA 有不同的归纳偏置。在某个具体数据上 LSTM 表现更好，不意味着在所有音频上都更好。而且 LSTM/Transformer 的泛化能力受训练数据量限制。

### ❌ "某个维度最难学习说明数据有问题"

有些维度（如复杂度）天然就接近随机，这是音频的特性，不是数据质量问题。

### ❌ "预测不准说明模型不好"

有些信号本身就是不可预测的（白噪声）。这时模型预测误差大 = 模型诚实地表示"我无法找到规律"——这是一种正确的行为。
